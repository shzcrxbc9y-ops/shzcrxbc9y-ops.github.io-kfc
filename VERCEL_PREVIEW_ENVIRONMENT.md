# Настройка предпроизводственной среды в Vercel

## Что такое предпроизводственная среда?

Предпроизводственная среда (Preview Environment) - это автоматически создаваемая среда для тестирования изменений перед деплоем в продакшн. Обычно она создается автоматически для каждого Pull Request.

## Нужно ли создавать вручную?

**НЕТ, обычно не нужно!** Vercel автоматически создает preview окружения для каждого PR.

Но если вы хотите создать именованную среду (например, "staging"), заполните форму так:

---

## Заполнение формы:

### Название окружающей среды (Environment Name):
```
staging
```
или
```
preview
```
или
```
development
```

### Описание (по желанию):
```
Среда для тестирования изменений перед продакшн деплоем
```

### Место для всех этапов развертывания:
- ✅ **Включите** эту опцию, если хотите, чтобы все развертывания шли в эту среду
- ❌ **Отключите**, если хотите использовать автоматические preview окружения

### При включении каждое квалифицированное слияние генерирует развертывание:
- ✅ **Включите**, если хотите автоматические деплои при слиянии веток
- ❌ **Отключите**, если хотите контролировать деплои вручную

### Автоматически назначайте домен новейшему развертыванию:
- ✅ **Включите** - всегда будет доступен последний деплой
- ❌ **Отключите** - нужно будет выбирать конкретный деплой

### Прикрепить переменные текущей среды из другой среды:
- Выберите **Production** - чтобы скопировать переменные окружения из продакшн
- Или оставьте пустым и добавьте переменные вручную

---

## Рекомендация:

**Для большинства проектов НЕ нужно создавать именованную среду вручную.**

Vercel автоматически создает preview окружения для каждого Pull Request. Это удобнее и проще.

Просто:
1. ✅ Добавьте переменные окружения для **Preview** (в разделе Environment Variables)
2. ✅ Создайте Pull Request в GitHub
3. ✅ Vercel автоматически создаст preview окружение

---

## Если все же хотите создать:

**Название:** `staging`  
**Описание:** `Staging environment for testing`  
**Место для всех этапов:** ❌ Отключить  
**Автоматические деплои:** ✅ Включить  
**Автоназначение домена:** ✅ Включить  
**Переменные:** Скопировать из Production
